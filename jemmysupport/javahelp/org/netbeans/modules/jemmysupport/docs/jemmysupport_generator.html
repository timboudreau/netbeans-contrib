Jemmy Component Generator

First select destination filesystem and package for generated component sources.

Create Screenshot checkbox gives you an ability to create screen shots of generated components. Images are stored in the same directory with the same name as generated component and with extension .png.

Show Component Editor checkbox enables/disables Component Editor appearing during process of generation. It allows on-line editing of grabbed structure before generated sources are finalized.

Merge Conflicts checkbox shows Merge Window in case of conflict with already present component source. Than you can control process of merging source differences.
Filenames of newly generated sources are numbered to avoid conflicts when Merge Conflicts checkbox is disabled.

Use Component Name checkbox enables that names of components are used to their identification

Start/Stop Component Generator button starts/stops thread waiting for some component to be focused and CTRL-F12 key to be pressed.
This button is enabled only when some package with write access is selected.
Component Generator is automatically stopped when its dialog is closed.

When generator is started and you see information message above Stop button you can invoke and select the component to be grabbed.
Selected components are highlighted by red border. Process of generation can be started only by pressing CTRL-F12 key. Component Editor dialog or Merger is displayed then in dependency of previous selection.

Do not access selected component till "Please wait, processing..." message is displayed !

Component Editor

Component Editor displays tree of known visible sub-components under selected component on the left and properties of selected sub-component in property sheet on the right.

Any sub-component node can be deleted from tree by pushing Delete key or by selecting Delete from its popup menu.

Sub-component properties:

interanlLabels property is list of text labels found inside sub-component.
These labels are used to generate public static final String fields for some operators (JList, JComboBox...).

shortName property is used mostly for low level method naming in generated source.

uniqueName property is used for unique sub-component identification, for inner fields and sub-component getters naming in generated source.
Root nodes uniqueName represents its class and file name without extension.

Note: uniqueName property is changed in dependency of shortName property to simplify modifications.
