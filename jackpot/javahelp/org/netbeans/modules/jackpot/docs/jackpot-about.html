About Jackpot

See Also

The Jackpot module adds reengineering capabilities to the NetBeans IDE. Reengineering is similar to refactoring, but focuses on making specific global changes to a whole source base. Jackpot works by building a model of one or more projects, then searching for patterns and applying transformations to those patterns in a safe manner. For example, Jackpot can be used to translate deprecated methods to their recommended replacements, all in one pass.

Jackpot has a rule language which can be used to specify transformation patterns of Java statements and expressions. This language is similar to Java, but has additional keywords and extensions to support wildcard matching. The Jackpot module includes support for creating, editing, and executing these rule files in the IDE. Rule files can be distributed independently from the IDE and then either executed manually within it or installed as a menu command.

Jackpot transformations of all kinds can be written in Java. The is built upon the new Tree API in Java 6, extending it to support source tree modifying and writing those changes back to source code. The Jackpot module uses the NetBeans Java Source module to locate and edit project source files.
