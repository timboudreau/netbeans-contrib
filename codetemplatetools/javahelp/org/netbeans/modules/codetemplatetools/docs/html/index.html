Code Template Tools

Features

  • Interactively create, modify or delete code templates
  • Interactively insert code templates
  • Cut, Copy and Paste to Register 0 through 9 and A through Z
ActionKey bindings
Expand TemplateCtrl+J SPACE
Surround With...Ctrl+J T
Cut to RegisterCtrl+J R X
Copy to RegisterCtrl+J R C
Paste from RegisterCtrl+J R V

How to Use

To create a Code Template from selection in the editor

  • Select text in editor
  • Invoke Create Template... action
  • Name the template and insert any parameters using the ${parameter-name} syntax
  • Supports additonal parameters:
    • ${selection} - If the text editor had any selection, that selection will be inserted at the location of ${selection} parameter. Use ${selection editable=false} to prevent editing of inserted text.
    • ${clipboard-content} - If the clipboard had any text, that text will be inserted at the location of ${clipboard-content} parameter. Use ${clipboard-content editable=false} to prevent editing of inserted text
    • ${input-parameterName prompt="prompt string" defaultValue="value1" valuesSeparator="," enumeratedValues="value1,value2,value3" suggestedValues="value1,value2,value3"} - The user will be be prompted to enter the value of the parameter. The parameter name part (excluding the input- prefix is used as the default prompt and default value. The values of input-... parameters can be strictly constrained using the enumeratedValues hint. The values of input-... parameters can be loosely constrained using the suggestedValues hint. The values separator can be specified using valuesSeparator hint.
  • Click on Save to save the Code Template

To insert a Code Template

  • Invoke Show Templates... action
  • Select a template from the list and preview it
  • Click on Insert to insert the selected Code Template

NOTE: The templates which use the ${selection} parameter are shown with icon.

Surround With Action

The Surround With Action shows all the templates in a dialog. You can select a template and type ENTER or double click to insert that template. If the editor has a selection the templates that make use of the ${selection} parameter are shown at the front of the list.

To create a Code Template

  • Invoke Show Templates... action
  • Click on New... to create a template.

To expand a Code Template

  • Invoke Expand Template action (Ctrl+J SPACE)

With caret after the word matching the Code Template name, invoking the Expand Template action expands the Code Template.

To modify or delete a Code Template

  • Invoke Show Templates... action
  • Select a template from the list and preview it
  • Click on Modify... to modify the selected template
  • Click on Delete... to delete the selected template

To import Code Templates

  • Invoke Show Templates... action
  • Click on Import... to import code templates
  • Select the xml file containing code templates. The xml file format is same as the .netbeans/<nb-version>/config/Editors/<mime-type>/CodeTemplates/org-netbeans-modules-editor-settings-CustomCodeTemplates.xml. For example:
    • <?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE codetemplates PUBLIC "-//NetBeans//DTD Editor Code Templates settings 1.0//EN" "http://www.netbeans.org/dtds/EditorCodeTemplates-1_0.dtd">
<codetemplates>
    <codetemplate abbreviation="com" xml:space="preserve">
        <code>/* */</code>
    </codetemplate>
</codetemplates>

To use Registers

  • Select text in editor
  • Invoke Cut to Register... ( Ctrl+J R X ), select the Register from the dialog and click Ok. The selection is cut to selected Register.
  • Invoke Copy to Register... ( Ctrl+J R C ), select the Register from the dialog and click Ok. The selection is copied to selected Register.
  • Invoke Paste from Register... ( Ctrl+J R V ), select the Register from the dialog and click Ok. The contents of the selected Register are inserted as if it were a template. That way the magic of parameter substitution can happen.

The Registers are implemented as Code Templates with name REG_ + registerName. The REG_ can be overridden using the System property -Dorg.netbeans.modules.codetemplatetools.REGISTER_PREFIX=prefix.
