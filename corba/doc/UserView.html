6. CORBA Naming System Browser

CORBA Naming System Browser allows user to add new naming context, browse them, and generate code usable in developing server or client that use Naming Service as bind method.

6.1 Code Generation

     User is able to generate lookup code to Clipboard from each Object in  the   directory context hierarchy or binding code from Context objects by  selecting   the desired node and choosing the copy action from pop up menu.  If the node represents NamingContext, than the user is able to copy binding  code, else if the node represents bound object the access code for this  object. After pasting the code to the instance of server or client template  user obtains fully functional code for manipulating with objects in Naming  Service.

6.2 Object Properties

    Browser also offers to user possibility to display detailed information  about object by selecting the corresponding node and choosing Properties  action form pop up menu. In these properties user can see the IOR (Inter operable  Object Reference) of the object, id (name displayed in browser) of object  and kind of object.

6.3 Manipulation Function

   CORBA Naming System Browser has also functions for creating new Naming Contexts, binding new Objects or Contexts and unbinding them. These functions can be accessed from  pop up menu.
   When action for creating new context is activated the dialog appears. In this dialog user fills id and kind of new context.
    By activating bind new Object or Context action similar dialog appears. In this dialog user fills id, kind and IOR or URL of resource containing the IOR.


7. CORBA Interface Repository Browser

CORBA Interface Repository Browser allows user to add new interface repositories,  browse them, explore metadata contained in them and generate IDL (Interface  Definition Laguage) from them. The interface repository browser is also  used by CORBA Wizard to allow user import IDL definition directly from IR.

7.1 Code Generation

   User is able to generate IDL source code for Interface Repository mounted in  the browser or for its reasonable part. The code generation action is not allowed on nodes that can not be generated separately and must be generated only as a member of some container, e.g. attribute or operation. The code is generated into Clipboard and than can be pasted to target idl file. The Copy Code action can be started by selected desired node and choosing the action from pop up menu.

7.2 Object Properties

      Browser also offers to user posibility to explore properties of objects  in Interface Repository. The properties contains for example RepositoryId  of IDL type, name, version and proprietary information that depends on kind  of selected object, for example  for interface these information are: base  interfaces.

8. CORBA Wizard

    CORBA Wizard helps user to develop complete CORBA application. It leads  user through all stages of development of CORBA application, namely creating  or importing IDL, generating implementation files, creating of server and  or client.

8.1 Choosing Destination Package

   The first panel that appears in the CORBA Wizard lets user to select destination package in mounted file systems and specify the name of IDL file. From the name of IDL file will be also derived the names of client and server. If the specified IDL file already exists in the destination package the next button is disabled.

8.2 Choosing Type of CORBA Application

   In the second panel user chooses which part of CORBA application would like to create. If he wants to import IDL file or create it. If he wants to generate implementation files, create client and or server.

8.3 Choosing ORB Implementation

   The third panel lets user to specify which ORB Implementation and binding method he would like to use. It also lets him specify if he wants to generate inheriance based or tie based implementation files.

8.4 Specifying IDL

   The fourth panel is for creating ir importing IDL, its version depends on the value of Create IDL item in the second panel.

8.4.1 Creating IDL

  This panel allows user to create IDL file using writable tree. The new IDL entries are added by selecting parent container in tree and pressing button that represents the desired data type in dynamic button panel. After the buton is clicked the dialog appers, in this dialog user fills information about IDL entity.

8.4.2 Importing IDL

   This panel is design to import IDL either from file or from Interface Repository.


8.5 Importing IDL

      The last panel informs user about finishing of CORBA Wizard steps, by pressing the Finish button wizard generates requested source code.
